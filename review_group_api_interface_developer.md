# 视频审核工具分组审核接口文档（开发对接版）

## 1. 文档目的

本文档面向需要从“视频生成应用”或其他外部程序中，**直接启动视频审核工具并自动勾选指定审核分组**的开发者。

当前接口形态为：
- **命令行启动参数接口**
- 外部应用通过启动审核工具进程时传参
- 审核工具在启动阶段解析分组参数，并自动设置“请选择审核分组”中的勾选状态

该接口适用于以下场景：
- 生成工具在某一批视频生成完成后，直接跳转到审核工具
- 外部应用只希望审核某几个产品分组
- 需要一次性传入多个分组，减少人工点击筛选

---

## 2. 接口能力概述

审核工具支持以下三种传参方式：

### 2.1 单个分组重复传参

```bash
video_review_tool.exe --review-group "266渐变" --review-group "301黑色"
```

适合调用端已有数组列表时逐个拼接参数。

### 2.2 一次传入多个分组

```bash
video_review_tool.exe --review-groups "266渐变,301黑色,420白色"
```

支持以下分隔符：
- 英文逗号 `,`
- 英文分号 `;`
- 竖线 `|`
- 换行符

### 2.3 通过文件传入多个分组

```bash
video_review_tool.exe --review-groups-file "F:\temp\review_groups.txt"
```

文件支持两种内容格式：

#### 纯文本格式

```txt
266渐变
301黑色
420白色
```

#### JSON 数组格式

```json
["266渐变", "301黑色", "420白色"]
```

---

## 3. 分组定义规则

审核工具中的“分组”不是任意标签，而是根据 `分镜生成` 目录下的视频所属文件夹名自动抽取。

### 3.1 抽取规则

对于 `分镜生成` 下的一级分组文件夹，取：
- **第一个 `-` 前的内容** 作为分组名

例如：

| 文件夹名 | 识别出的审核分组 |
|---|---|
| `266渐变-英国` | `266渐变` |
| `266渐变-德国` | `266渐变` |
| `301黑色-美区` | `301黑色` |
| `420白色` | `420白色` |

### 3.2 接口传值要求

调用端传入的分组名，必须与工具内部识别出的分组值一致。

也就是说，如果你的目录是：

```text
分镜生成/
  266渐变-英国/
  266渐变-德国/
  301黑色-美国/
```

那么应传：
- `266渐变`
- `301黑色`

而不是传完整文件夹名：
- `266渐变-英国`
- `266渐变-德国`

---

## 4. 参数说明

### 4.1 `--review-group`

#### 类型
字符串，可重复出现。

#### 说明
每出现一次，表示追加一个待审核分组。

#### 示例

```bash
video_review_tool.exe --review-group "266渐变" --review-group "301黑色"
```

#### 适用场景
- 上游程序内部已经有 `list[str]`
- 需要避免自己做分隔符拼接

---

### 4.2 `--review-groups`

#### 类型
字符串，单次传入多个分组。

#### 说明
工具会自动按分隔符拆分。

#### 支持分隔符
- `,`
- `;`
- `|`
- `\n`

#### 示例

```bash
video_review_tool.exe --review-groups "266渐变,301黑色,420白色"
```

```bash
video_review_tool.exe --review-groups "266渐变|301黑色|420白色"
```

#### 适用场景
- 上游程序更方便传一个字符串
- 从输入框、配置项、文本框中直接带参

---

### 4.3 `--review-groups-file`

#### 类型
文件路径。

#### 说明
工具启动后读取该文件，解析出多个分组。

#### 支持格式
- 纯文本：一行一个分组
- JSON 数组：`["A", "B", "C"]`

#### 示例

```bash
video_review_tool.exe --review-groups-file "F:\temp\review_groups.txt"
```

#### 适用场景
- 分组很多
- 由上游程序先落地一个文件再调用审核工具
- 需要便于排查和复现启动参数

---

## 5. 参数优先级与合并规则

审核工具会把以下三类来源的分组合并：
- `--review-group`
- `--review-groups`
- `--review-groups-file`

### 5.1 合并规则

1. 所有来源统一汇总
2. 自动去重
3. 去除空值
4. 最终形成本次启动的目标分组集合

### 5.2 优先级说明

这三类参数不是互斥关系，而是**合并关系**。

例如：

```bash
video_review_tool.exe ^
  --review-group "266渐变" ^
  --review-groups "301黑色,420白色" ^
  --review-groups-file "F:\temp\extra_groups.txt"
```

最终勾选集合 = 三部分求并集。

---

## 6. 审核工具启动后的行为

### 6.1 有传分组参数时

工具启动后会：

1. 扫描 `分镜生成` 目录
2. 根据目录名抽取可审核分组
3. 将传入分组与实际分组做匹配
4. 自动勾选匹配成功的分组
5. 队列中只加载这些分组下的视频

### 6.2 没有传分组参数时

保持工具原有行为：
- 使用界面当前保存的分组选择状态
- 若是首次启动，则默认全选

### 6.3 传入分组不存在时

如果传入的某个分组在当前 `分镜生成` 中没有匹配到：
- 不会报错退出
- 不会阻断程序启动
- 会在日志中提示该分组未命中

---

## 7. 运行结果与返回行为

这是一个**进程启动型接口**，不是 HTTP 接口，因此没有标准 JSON 返回体。

### 7.1 启动成功的定义

调用端成功拉起审核工具进程，即视为接口调用成功。

### 7.2 分组是否命中

当前版本中，分组命中结果体现在：
- 审核工具界面的勾选状态
- 审核工具右侧日志

### 7.3 建议调用端如何判断成功

建议外部程序按以下层级判断：

#### 一级：进程是否成功启动
- `subprocess.Popen(...)` 未抛异常
- 说明 exe 已正常拉起

#### 二级：是否需要进一步校验分组命中
如果你的上游程序需要更强确认，可以增加一层业务校验：
- 调用前，自己先拿待传分组与生成目录做一次映射确认
- 或后续让审核工具再落一份状态文件

> 当前版本没有单独提供“启动后返回命中结果文件”的机制。

## 8. 推荐对接方式

### 8.1 Python 调用示例：多次 `--review-group`

```python
import subprocess

exe_path = r"F:\r\视频审核工具.exe"
groups = ["266渐变", "301黑色"]

cmd = [exe_path]
for g in groups:
    cmd.extend(["--review-group", g])

subprocess.Popen(cmd)
```

### 8.2 Python 调用示例：一次性 `--review-groups`

```python
import subprocess

exe_path = r"F:\r\视频审核工具.exe"
group_text = "266渐变,301黑色,420白色"

subprocess.Popen([exe_path, "--review-groups", group_text])
```

### 8.3 Python 调用示例：通过文件传递

```python
import json
import subprocess
from pathlib import Path

exe_path = r"F:\r\视频审核工具.exe"
tmp_file = Path(r"F:\temp\review_groups.json")
groups = ["266渐变", "301黑色", "420白色"]

tmp_file.write_text(json.dumps(groups, ensure_ascii=False), encoding="utf-8")
subprocess.Popen([exe_path, "--review-groups-file", str(tmp_file)])
```

### 8.4 Windows CMD 示例

```bat
"F:\r\视频审核工具.exe" --review-group "266渐变" --review-group "301黑色"
```

## 9. 推荐接入时序

适合生成工具接入的推荐流程如下：

1. 生成工具完成视频生成
2. 生成工具根据本次生成任务，整理出涉及到的产品分组
3. 调用审核工具 exe，并将分组通过参数传入
4. 审核工具启动后自动勾选对应分组
5. 用户直接开始审核

### 9.1 示例时序

```text
生成应用完成任务
    ↓
收集本次涉及分组
    ↓
启动审核工具并传入分组参数
    ↓
审核工具自动筛选对应队列
    ↓
用户开始审核
```

## 10. 异常与边界情况

### 10.1 传入空分组

如果传入参数为空、全空格、拆分后无有效值：
- 视为未传分组参数
- 工具按自身默认/上次保存状态运行

### 10.2 分组名重复

例如：

```bash
video_review_tool.exe --review-group "266渐变" --review-group "266渐变"
```

结果：
- 工具内部会自动去重
- 不会重复勾选或报错

### 10.3 传入完整文件夹名而不是分组名

例如传入：
- `266渐变-英国`

但工具分组是：
- `266渐变`

结果：
- 大概率视为未命中
- 建议上游程序始终传“第一个 `-` 前的值”

### 10.4 `分镜生成` 目录不可访问

如果源目录不存在、共享盘断开或无权限：
- 工具会启动
- 但无法正确加载队列
- 分组筛选也无法命中
- 界面会提示目录不可用

### 10.5 文件传参路径不存在

如果 `--review-groups-file` 指向的文件不存在或读取失败：
- 不应导致工具整体崩溃
- 该来源的分组会被忽略
- 其他参数来源仍可继续生效

## 11. 对接建议

### 11.1 最推荐方式

对大多数生成工具，推荐优先使用：

```bash
--review-groups
```

原因：
- 最简单
- 参数结构清晰
- 不需要额外写文件

### 11.2 分组很多时

如果一次要传几十个甚至更多分组，推荐：

```bash
--review-groups-file
```

原因：
- 命令行更短
- 更好排查
- 不容易受转义影响

### 11.3 上游程序要自行做一次分组归一化

建议生成端在传值前先做一次：
- 去首尾空格
- 去重
- 统一使用第一个 `-` 前的值

例如：
- `266渐变-英国` → `266渐变`
- `266渐变-德国` → `266渐变`

## 12. 后续可扩展方向

如果你后续需要更强的程序对接能力，可以继续扩展为：

### 12.1 JSON 字符串参数
例如：

```bash
video_review_tool.exe --review-groups-json "[\"266渐变\",\"301黑色\"]"
```

### 12.2 状态回写文件
工具启动后输出一份：
- 实际命中的分组
- 未命中的分组
- 当前加载的视频数量

适合上游程序做更强校验。

### 12.3 本地 HTTP 接口
由审核工具监听本地端口，生成工具直接 POST 分组列表。

适合：
- 审核工具常驻
- 多次下发审核任务
- 不想重复拉起 exe

---

## 13. 最简接入示例

如果你只想最快接上，推荐直接这样：

```python
import subprocess

exe_path = r"F:\r\视频审核工具.exe"
groups = ["266渐变", "301黑色"]
subprocess.Popen([exe_path, "--review-groups", ",".join(groups)])
```

这样你的生成应用在视频生成结束后，只要整理出本次涉及的产品分组，就能一键拉起审核工具并自动筛到目标队列。
